# Issue: 図表の配色と本文の強調 ― 単調さの解消

対象: Codex
種別: デザイン改善（配色 / 強調 / 視覚的メリハリ）
優先度: 中〜高

---

## 0. 結論

中身は良いが**見た目が単調**。原因は2つ。

1. **図表がほぼ `--accent`（ティール）一色**で、意味の違い（起点/終点、強調すべき箇所、層の役割）を色で表現していない。
2. **本文に一切の強調が無い**（`<strong>/<mark>` が全ページ0個）。重要語が地の文に埋もれる。

学習サイトとして「どこが大事か」が一目で分かるよう、**色と強調に“意味”を持たせる**。
ただし無意味な装飾色は足さない（過去の「ティール/ゴールド交互の棒」のような、データに無関係な色分けはNG）。色は**意味かブランド識別のどちらか**を担うこと。

> 実装対象: [`assets/js/visualizations.js`](../assets/js/visualizations.js)（図表描画）、[`scripts/generate-site.js`](../scripts/generate-site.js)（本文の強調生成）、[`assets/css/styles.css`](../assets/css/styles.css)（配色トークン）。
> HTMLは生成物。`node scripts/generate-site.js` で再生成する。

## 2. 図表の配色改善

### 2-1. セマンティックな配色トークンを定義

[`assets/css/styles.css`](../assets/css/styles.css) の `:root` / `[data-theme="dark"]` に、**役割ベース**の色を追加（既存 `--accent/--gold/--coral/--ink` を活用しつつ命名）。例:

```css
:root {
  --viz-start:  var(--gold);          /* 起点・入力 */
  --viz-step:   var(--accent);        /* 通常ステップ・処理 */
  --viz-key:    var(--coral);         /* 強調・終点・結論 */
  --viz-accent2:#3b82c4;              /* 第2系統（対比用） */
  --viz-muted:  var(--surface-strong);/* 非アクティブ */
}
```

- **ダークテーマでも同等のコントラスト**になるよう `[data-theme="dark"]` 側も定義。
- 数は絞る（4〜5色）。増やしすぎは逆に散らかる。

### 2-2. フロー図（最優先）

[`renderFlow`](../assets/js/visualizations.js#L323) を改善:

- **起点**＝`--viz-start`、**終点（結論/判定）**＝`--viz-key`、中間＝`--viz-step`(薄め) で塗り分ける。
- 各箱に**連番バッジ**（①②③…）を付け、順序を明確化。
- コネクタを**矢印（▶ / arrowhead）**にして方向を示す。
- `spec` に「強調ステップ」を指定できるようにし（例 `emphasis: index`）、該当箱を太枠＋濃色でハイライトできるようにする。テーマ上の山場（記号接地なら「経験との対応」「意味形成」）を強調可能に。
- ステップ文字が小さい問題も合わせて改善（フォントサイズ・箱サイズ）。

### 2-3. ネットワーク図

[`renderNetwork`](../assets/js/visualizations.js#L286): 層の役割で色分けを復活。
- 入力層＝`--viz-start`、隠れ層＝`--viz-step`、出力層＝`--viz-key`。
- 凡例（入力/隠れ/出力の色対応）を図内に表示。

### 2-4. 比較表/対応表（SVG/HTML両方）

- **キー列（先頭列）**を他列と差別化（背景 `--accent-soft` など）。
- 「最重要行」や「試験で問われる行」を指定できれば、淡いハイライトを付けられるように（任意）。
- ヘッダ・ストライプ・ホバーで視認性を上げる。

### 2-5. 数値系グラフ（強調を色で）

- 回帰 [`renderRegression`](../assets/js/visualizations.js#L140): 回帰直線を主役色（`--viz-key`）、データ点は控えめ色、残差線は淡色。主役と脇役の対比を明確に。
- 分類 [`renderClassification`](../assets/js/visualizations.js#L200): 決定境界を強調色に。可能なら **TP/FP/FN/TN を4色 or 形で区別**し、凡例を付ける（現状はクラス色＋透明度のみで読み取りにくい）。
- 分布 [`renderDistribution`](../assets/js/visualizations.js#L88): 平均線を強調色で（現状コラルでOK）。ヒストグラムは1色でよいが、平均±1SDの帯を淡色で示すと「ばらつき」が直感的になる（任意）。

### 2-6. 共通ルール（重要）

- **色だけに意味を載せない**（アクセシビリティ）。色＋ラベル/番号/形を併用する。
- 色が意味を持つ図には**凡例**を付ける。
- WCAGのコントラストを確保（特にダークテーマ）。
- すべてCSS変数経由。ハードコード禁止（現状 [L181 `--ink`](../assets/js/visualizations.js#L181) 等は変数化済み、この方針を維持）。

## 3. 本文・テキストの強調

### 3-1. 重要語のハイライト

- レッスンの**キーワード（`item.terms`）**を本文中で自動的に強調する。
  [`generate-site.js`](../scripts/generate-site.js) の本文生成時に、用語に一致する箇所を `<strong class="term">` でラップ（最初の出現のみ等、過剰強調を避けるルールを設ける）。
- さらに、著作データ側で**明示的な強調**を書けるようにする。
  `content/lesson-content.json` の `explanation` で `**...**`（または `[[強調]]`）記法を許可し、生成時に `<mark class="hl">` へ変換。
  → 「ここが試験で問われる」という箇所を著者が指定して光らせられる。

### 3-2. 強調のスタイル（控えめに）

`styles.css` に:
```css
.term { color: var(--accent-strong); font-weight: 700; }
mark.hl { background: var(--accent-soft); color: var(--text); padding: 0 .15em; border-radius: 3px; }
```
- 乱用しない。1段落に強調が多すぎると逆効果。**1段落に1〜2か所**を目安に。

### 3-3. コールアウト（注意/ポイント/頻出）

単調な段落の連続を崩すため、**コールアウト枠**を導入。
- 種別: `ポイント` / `試験で頻出` / `注意` / `具体例` など。
- 左ボーダー＋淡い背景＋アイコン的ラベルで色分け（種別ごとに `--accent / --gold / --coral`）。
- 著作データから種別とテキストを与えられるようにし、生成時に
  `<aside class="callout callout--exam">…</aside>` 等で出力。
- 例（学習効果が高い）: 各レッスンに「試験で問われる観点」を1つコールアウトで置く。

## 4. サイト全体のメリハリ（アドバイス）

> ここは私（レビュア）からの提案。必須ではないが効果が高い順。

1. **章ごとのアクセントカラー（ブランド識別）**: 10章それぞれに淡いアクセント色を割り当て、章インデックス・パンくず・タグ・見出し下線などに反映。サイト全体が一色で塗られた印象を解消し、「今どの章にいるか」も伝わる。
   - 例: 法律=青系、倫理=紫系、数理=緑系…。彩度は抑え、本文可読性は崩さない。
   - 実装: `chapters/<slug>/` に `data-chapter="<slug>"` を付け、CSSで章別 `--accent` を上書き。
2. **タグ/カテゴリのカラーコード**: 用語タグ（`.tag`）やカテゴリchipを単色から、種別ごとの淡色へ。
3. **セクション見出しのアクセント**: `h3` に短いカラーバー/下線を付け、視線の区切りを作る。
4. **進捗・正誤の色は既存の意味色を踏襲**（緑=正解/完了、赤=不正解、ゴールド=強調）。新しい色を足すときは既存の意味と衝突させない。
5. **やりすぎ注意**: 色は「意味」か「章の識別」に限定。装飾だけの多色化はしない。ダークモードでの破綻と、色覚多様性（色だけに頼らない）を常に確認。

## 5. 受け入れ基準

### 全体
- [ ] サイトが一色塗りの印象から脱し、章やセクションにメリハリがある。
- [ ] 色覚多様性・コントラストに配慮（色のみで情報を伝えない）。
- [ ] `node scripts/generate-site.js` 後にHTMLへ反映。HTML直編集はしない。

## 7. やってはいけないこと

- データに無関係な装飾だけの多色化（過去の「交互色の棒」の再来）。
- 色だけで正誤・順序・役割を伝える（アクセシビリティ違反）。
- 強調の乱用（全部太字＝強調なしと同じ）。
- ハードコード色（CSS変数を使う）。
- HTML直編集（再生成で消える）。

# G検定インタラクティブ学習サイト構成プラン

## Summary

- `deep-research-report.md` と JDLA公式ページを基準に、GitHub Pages で公開できる静的HTML学習サイトを作る。
- 構成は JDLA公式の試験範囲に合わせた **10学習モジュール + 横断復習** にする。
- 初期公開版はバックエンドなし、ログインなし、個人情報保存なし。`index.html` と `assets/` 配下の CSS/JS だけで動く構成にする。
- 公式本文や例題の大量転載は避け、公式シラバスの見出しに沿った独自解説・独自問題として作る。
- 参照元: [JDLA G検定とは](https://www.jdla.org/certificate/general/)。

## 公開URL

https://masaspc.github.io/Gken/

## Key Changes

- トップページはランディングページではなく、すぐ学習を始められる **学習ダッシュボード** にする。
- 主要セクション:
  - `学習マップ`: 10分野の進捗、推奨順、弱点表示
  - `今日の学習`: 1モジュール分の短い導線
  - `インタラクティブ図解`: 回帰、分類、分布、ニューラルネットワーク、Attention など
  - `ミニテスト`: 多肢選択、数値入力、並べ替え、図操作課題
  - `用語集`: 検索可能なG検定キーワード辞典
  - `参照資料`: JDLA公式、官公庁資料、利用ライブラリ、更新日

## Implementation Plan

ファイル構成はシンプルな静的サイトにする。

- `index.html`: 全体UIと学習ダッシュボード
- `assets/css/styles.css`: レスポンシブデザイン、ダーク/ライト対応
- `assets/js/app.js`: ナビゲーション、進捗、状態管理
- `assets/js/visualizations.js`: グラフ、スライダー、動的図解
- `assets/js/quizzes.js`: 問題データ、自動採点、フィードバック
- `docs/sources.md`: 参照元と更新確認メモ
- `.nojekyll`: GitHub Pages の静的配信を安定させる設定

## デザイン方針

- モダンで公開品質の日本語UIにする。
- 最初の画面に学習マップ、進捗、開始ボタン、代表的な可視化を表示する。
- スマホ、タブレット、PCで破綻しないレスポンシブレイアウトにする。
- 装飾より学習効率を優先し、カード乱用や派手すぎる配色は避ける。
- ライトテーマとダークテーマを切り替えられるようにする。

## インタラクティブ要素

- 分布: 平均・分散・サンプル数スライダー
- 回帰: ノイズ、傾き、切片を変えて残差とMSEを表示
- 分類: 閾値を変えて混同行列、適合率、再現率を表示
- ニューラルネットワーク: 層数・活性化関数を切り替えて構造図を更新
- Attention: トークン間の重みをヒートマップで表示

## Test Plan

### ローカル確認

- `index.html` をブラウザで開いて表示崩れがないこと。
- 各モジュールへ移動できること。
- スライダー操作でグラフと数値が即時更新されること。
- ミニテストの採点結果が正しく出ること。

### 表示確認

- PC幅、タブレット幅、スマホ幅で確認する。
- 日本語テキストがボタンやカードからはみ出さないこと。
- キーボード操作で主要ボタン・入力欄に到達できること。

### 公開確認

- GitHub Pages のURLでCSS/JSが読み込まれること。
- 相対パスが壊れていないこと。
- README から公開サイトへリンクできること。

## Assumptions

- 「制的HTML」は「静的HTML」の意味として扱う。
- 初期版は1ページ完結型の静的サイトにして、完成度を優先する。
- 公式資料は参照・要約に留め、JDLA公式文面や例題の大量転載はしない。
- 学習進捗は初期版ではブラウザの `localStorage` に保存する。
- 会員登録、サーバー保存、LMS連携、Python実行環境は初期公開範囲に含めない。
